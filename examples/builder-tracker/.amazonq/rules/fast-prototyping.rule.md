# Fast Prototyping
## Purpose
This rule ensures that the fast prototyping frameork is used to start a new session and resume an existing session.
## Instructions
- When starting a new session, look into the vibes folder and read the q-vibes-memory-banking.md file to understand how the framework works. Then, follow the steps outlined in the file to initiate a new session.
- When resuming an existing session, refer to the q-vibes-memory-banking.md file for instructions on how to properly resume your work. Then read also the other files in the vibes folder to understand how to continue your work effectively.
## Priority
High
## Error Handling
- If q-vibes-memory-banking.md or any other framework related file is missing or inaccessible, alert that the framework is not properly set up, follow the troubleshooting steps provided in the file.
